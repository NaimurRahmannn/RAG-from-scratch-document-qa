RAG stands for Retrieval-Augmented Generation.

RAG is a technique where an AI application first searches for relevant information from a document or database, then sends that information to a language model so the model can answer using that context.

A basic RAG system has several steps: document loading, text splitting, embedding generation, vector search, retrieval, prompt building, and answer generation.

Embeddings are numerical representations of text. They help computers compare the meaning of different pieces of text.

Chunking means splitting a large document into smaller pieces. This is important because language models have context limits and we usually only need the most relevant parts of a document.

A retriever finds the most relevant chunks for a user question.

The final prompt contains the retrieved context and the user question. The language model should answer only using the provided context.

If the answer is not available in the provided context, the model should say it does not know based on the document.
